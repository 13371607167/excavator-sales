## 快速开始

### 1️⃣ 启动部署界面

双击项目根目录下的：
```
启动部署界面.command
```

### 2️⃣ 在浏览器中操作

浏览器会自动打开：http://localhost:3000

### 3️⃣ 修改代码并发布

1. 在代码编辑器中修改你的代码
2. 在部署界面中点击 **[刷新状态]** 查看变更
3. 选择 **[快速发布]** 或 **[版本发布]**
4. 等待 1-2 分钟，网站自动更新

## 版本管理

### 版本号规则

采用 **语义化版本号**：`v主版本.次版本.修订号`

- **主版本 (Major)**：不兼容的重大更新
  - 例如：v1.0.0 → v2.0.0
- **次版本 (Minor)**：新增功能（向后兼容）
  - 例如：v1.0.0 → v1.1.0
- **修订号 (Patch)**：bug 修复和小的改进
  - 例如：v1.0.0 → v1.0.1

### 版本类型选择

在版本发布中，可以选择：

- **Patch** - 小修复（自动递增修订号）
- **Minor** - 新功能（自动递增次版本）
- **Major** - 大版本（自动递增主版本）
- **Custom** - 自定义版本号

## 回滚到旧版本

### 方法一：通过 GitHub 界面（推荐）

1. 访问：https://github.com/13371607167/excavator-sales/releases
2. 找到你想要回滚的版本
3. 点击 **[Browse code]** 查看该版本的代码
4. 点击 **[main]** → **[Create branch]** 创建新分支
5. 合并新分支到 main

### 方法二：通过命令行

```bash
# 查看所有版本
git tag

# 查看某个版本的详细信息
git show v1.0.0

# 回滚到指定版本（会丢失之后的提交，请谨慎）
git reset --hard v1.0.0
git push --force origin main
```

⚠️ **警告**：强制推送会覆盖远程历史，请谨慎使用！

## 常见问题

### Q: 部署后网站多久更新？
A: 通常需要 1-2 分钟，GitHub Actions 会自动构建并部署。

### Q: 我想查看部署进度？
A: 访问：https://github.com/13371607167/excavator-sales/actions

### Q: 忘记创建版本号了，能补吗？
A: 可以！在本地创建：
```bash
git tag -a v1.0.1 -m "版本说明"
git push origin v1.0.1
```

### Q: 怎么查看当前版本？
A: 打开部署界面，在 **[当前状态]** 卡片中可以看到。
